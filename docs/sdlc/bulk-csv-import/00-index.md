**Slug:** `bulk-csv-import`
**Started:** 2026-05-04 (reference artifact)
**Owner:** Aurora data-platform squad
**Description:** Tenant admins upload a CSV of up to 50,000 accounts. The system validates each row, deduplicates against existing accounts, and processes asynchronously. The UI shows progress and a per-row outcome report. SOC 2 audit log captures every import.

## Top risks identified across stages

1. **CSV bomb / decompression risk** (stage 5) — addressed by `IMPORTS_MAX_CELL_BYTES=4096` cap.
2. **Tenant cross-contamination** (stage 5) — addressed by tenant_id always from JWT (never from CSV row), repo-layer scope checks, repository unit tests.
3. **Hot-tenant queue starvation** (stage 6) — addressed by per-tenant parallel-job cap of 1 + global worker concurrency 8.

## Top follow-ups for the team

1. **Decision needed**: outcome CSV retention. Proposal is 90 days; compliance to confirm against GDPR Art. 17.
2. **Tech debt logged**: timing-attack on dedup endpoint — implement constant-time comparison in v2.
3. **Stretch**: dry-run mode (`?dry_run=true`) for validation-only previews — defer to v2.
